|---|---|---|---|

| A-6 | Аудит всех операций с заявками и изменениями статусов | Банковское регулирование требует полной прослеживаемости. Критично для разрешения споров с клиентами. | Высокий |

| A-8 | Идемпотентность операций подачи заявок | Защита от случайного дублирования заявок при повторных запросах (network retry, двойной клик). | Высокий |
| A-9 | Версионирование API для интеграций | Обеспечит обратную совместимость при изменениях. Критично для партнёрского кол-центра. | Средний |
| A-10 | Централизованное логирование и мониторинг (ELK, Grafana) | Быстрая диагностика проблем в распределённой системе. Соответствие SLA 99.9%. | Высокий |

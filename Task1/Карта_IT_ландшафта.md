## IT-системы и технологии:

1. **АБС** - Delphi, Oracle, PL-SQL (внутренняя)
2. **Интернет-банк** - ASP.NET MVC, MS SQL (подрядчик)
3. **Сайт** - PHP, React.js (внутренняя)
4. **Система кол-центра** - React.js, Java Spring Boot, PostgreSQL (подрядчик)
5. **Кредитный конвейер** - Camunda BPM, Oracle, Java (подрядчик)
6. **Скоринг** - Python, Flask, PostgreSQL (внутренняя)
7. **БКИ** - внешняя, REST API
8. **СМС-шлюз** - внутренняя
9. **Система партнёрского кол-центра** - внешняя
10. **Excel** - ручное ведение ставок

## Процесс открытия депозита (текущий):

1. Клиент → Отделение/Кол-центр
2. Отдел кредитов → Excel (расчёт ставок на основе ЦБ)
3. Excel → Email → Бэк-офис депозитов (ежедневно)
4. Бэк-офис депозитов → АБС (определение ставки)
5. АБС → СМС-шлюз → Клиент (уведомление)
6. Клиент → Отделение (подтверждение)
7. Фронт-офис → АБС (создание депозита)

**Специальные ставки:** Бэк-офис депозитов → Email → Бэк-офис кредитов → АБС (анализ) → Email → Excel → Email → Фронт-офис

---

## Процесс открытия кредита:

1. Клиент → Отделение
2. Фронт-офис → АБС (создание заявки)
3. АБС → Кредитный конвейер (БД, раз в день)
4. Бэк-офис кредитов → Кредитный конвейер (обработка)
5. Кредитный конвейер → Скоринг (REST API)
6. Скоринг запрашивает:
   - АБС (данные клиента, платежи) - раз в сутки
   - БКИ (REST API) - онлайн
7. Скоринг → Кредитный конвейер (оценка)
8. Бэк-офис → Решение (одобрение/отказ)
9. Кредитный конвейер → АБС (финальный статус, раз в день)
10. АБС → Зачисление средств на счёт
